# PRD Grill Findings

### Finding 1 - Runtime Receipts Must Be The Authority

Status: resolved

The PRD now states that agent receipts remain advisory evidence and cannot satisfy runtime-sensitive claims. The authoritative receipts must be generated by the supervisor with `source=supervisor` and `evidence_grade=runtime_native`.

### Finding 2 - The Positive Path Needs Real Worktree State

Status: resolved

The PRD now requires a tiny real git repository in the positive test path so that acceptance depends on observed changed files, deliverable existence, and a supervisor test rerun instead of fixture-only receipt injection.

### Finding 3 - Reviewer Roles Stay Separate

Status: resolved

The PRD now keeps Cursor SDK and Claude Code review as downstream evidence reviewers. They cannot override a red deterministic runtime evidence probe.
