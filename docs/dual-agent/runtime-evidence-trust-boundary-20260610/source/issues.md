# Implementation Slices

## Slice 1: Receipt Provenance Sanitization

Priority: P1

Scope: Add shared provenance helpers, downgrade caller-stamped supervisor/runtime-native receipts at MCP and verification boundaries, and record downgrade events in the ledger.

Acceptance Criteria:

- [ ] Caller receipt with `source="supervisor"` becomes `source="caller_claimed_supervisor"` before claim verification.
- [ ] Caller receipt with `evidence_grade="runtime_native"` becomes `evidence_grade="self_reported"` before runtime-native checks.
- [ ] `receipt_provenance_downgraded` events include receipt id, claimed fields, effective fields, scope, and reasons.
- [ ] Verification helpers require current in-process runtime receipt ids for supervisor/runtime-native authority.

## Slice 2: Current-Gate Runtime Receipt Correlation

Priority: P1

Scope: Thread current runtime receipt ids from `collect_runtime_evidence` into `verify_workflow_claims` and `verify_gate_deliverable_evidence`, and ensure runtime receipts override same-id caller receipts during merge.

Acceptance Criteria:

- [ ] Honest execution and outcome gates still accept runtime evidence generated by the supervisor.
- [ ] Stale runtime receipts from earlier gates do not satisfy the current gate invocation.
- [ ] Same-id forged caller receipts cannot shadow the collector's in-process receipts.
- [ ] Existing workflow route and reviewer behavior remain unchanged.

## Slice 3: Declared Test Command Confinement

Priority: P1

Scope: Replace shell execution in runtime evidence with argv-form execution, parse an allowlist of pytest and make-test patterns, and reject arbitrary commands.

Acceptance Criteria:

- [ ] `python -c` declarations are rejected and not executed.
- [ ] Shell metacharacters and arbitrary command prefixes are rejected.
- [ ] Allowlisted pytest commands run with `shell=False` and report real pass or fail status.
- [ ] Rejected commands produce `runtime_test_command_rejected` in runtime evidence results.

## Slice 4: Validation Environment Hard Failure

Priority: P2

Scope: Build a scrubbed subprocess environment and classify unavailable pytest or missing executables as blocking runtime evidence failures.

Acceptance Criteria:

- [ ] Validation subprocess env excludes API keys, tokens, secrets, credentials, auth values, and passwords.
- [ ] The explicit validation interpreter is used, preferring the validation worktree `.venv` when present.
- [ ] Missing pytest is reported as `runtime_test_environment_unavailable`.
- [ ] The full workflow and runtime evidence test suites remain green.
